行政痛點

AI 生成的排版風格總是混亂不符公文規範？

每次叫 AI 產生 Word 或教材，標題字體大小、顏色都不一樣？每次都要重複下一大堆排版提示詞，既繁瑣又容易遺漏？

傳統做法
每次對話都得重新貼上：「大標要藍色、中文字型要標楷體、首行要縮排...」AI 聊到一半還會忘記。
技能包解法
將排版規範封裝在目錄下，呼叫 @doc_expert，AI 自動讀取手冊、寫代碼生成完美格式。

運作原理

技能包如何教導本地端 AI 大腦？

Custom Skill 是將專家規則封裝成靜態 Markdown 文件，AI 偵測到召喚後，在背景進行學習並注入代碼。

1. 定位與偵測
偵測對話中出現的標記 @doc_expert，在 .agents/skills/ 下找到 SKILL.md 檔案。
↓
2. 學習與注入
AI 自動調用 view_file 讀取手冊，並在背景寫 Python code 時將排版規範寫入程式中。

概念比較

對話 Prompt、System Prompt 與 Skill

三種不同的 AI 調教指令手段，定位各不相同，應根據情境選擇最省 Token、最專注的方案。

工具類型 一次性 Prompt System Prompt Custom Skill
記憶定位 臨時便利貼 核心人格與性格設定 專用任務操作手冊
優勢點 隨寫隨用，最靈活 自動常駐，安全防線 動態載入，不稀釋大腦專注度
最佳場景 一次性問答與微調 設定角色面具與安全性 Word排版、Excel大批次處理

推薦必裝

教師必備的三大開源專家技能包

透過開源社群或講師提供的專家包，讓 AI Agent 瞬間化身為專業文件工程師！

docx 技能包
Word 處理專家。教導 Agent 如何使用 python-docx 做複雜表格、頁首頁尾、灰色細線等排版。
xlsx 技能包
Excel 處理專家。教導 Agent 運用 openpyxl 對齊成績、繪製統計圖表、跨工作表登帳。
pdf 技能包
PDF 閱讀專家。教導 Agent 利用專屬程式套件精準讀取、摘要與分割複雜的 PDF 講義與考卷。

ONE-CLICK SETUP

課程懶人包：貼一行，自動設定好

課程懶人包 repo：
github.com/wukolo1206/antigravity-starter-kit

1. 下載 / clone 到本機：

# 終端機
git clone https://github.com/wukolo1206/antigravity-starter-kit.git

2. AntiGravity ➔ 新開專案 ➔ Add Folder 選這個資料夾。

3. 對它說：「請幫我完成課程設定。」

實作一

建立 @doc_expert 期末催繳排版技能包

在專案下建立 .agents/skills/doc_expert/SKILL.md，寫入標題深藍色粗體、中文縮排、淺灰分隔線與自動頁碼規範。

🚫 無 Skill 對照組

生成一般 Word

直接請 AI 將期末催繳資料寫成 Word，產出字型為普通新細明體，無段落縮排、無分隔線、無頁尾頁碼。

🌟 有 Skill 實驗組

呼叫 @doc_expert

對話提及載入 Skill，AI 自動遵循排版手冊寫代碼，雙擊開啟文件，排版格式一次完美到位！

實作二

複製並安裝「PDF 處理專家技能包」

將共享庫中的 pdf 資料夾，複製存檔到您工作目錄的 .agents/skills/pdf/ 中。

1. 安裝完成：確認 .agents/skills/pdf/SKILL.md 存在。

2. 召喚 @pdf：在對話框輸入：「載入 @pdf，幫我讀取本目錄下的 AI_Agent教師研習簡報.pdf，轉化為摘要大綱。」

3. 運作觀察：Agent 自動在背景閱讀 PDF 規範，呼叫 python 套件將 PDF 解析完成！

使用限制

必須掌握的安全防線與限制

技能包不是萬能魔法，在使用時有其特定的限制與安全檢查機制：

1. 讀檔工具依賴：AI 必須擁有讀取檔案工具（view_file）方可生效。

2. Token 預算限制：技能手冊不宜像整本書那樣長，會消耗大量 Token 並降低 AI 的專注度。

3. 靜態指令防禦：技能包本質是規則，無法讓 AI 跨越權限去做它本來就做不到的事情。
