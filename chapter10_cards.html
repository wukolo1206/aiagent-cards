準備工作

GCP 安全對接通行證 credentials.json

要讓本地程式與雲端 API 通訊，我們需要到 Google Cloud Platform (GCP) 申請免費金鑰。

1. 開啟 GCP 服務：在 GCP 控制台啟用 Drive, Slides, Forms, Sheets API。

2. 配置 OAuth 同意畫面：設定外部應用，新增您個人的測試 Google Email。

3. 下載金鑰 JSON：建立電腦版應用程式憑證，存為 credentials.json 並放在指定目錄。

首次登入拉起瀏覽器與產生 token.json

第一次執行程式時，會自動開啟瀏覽器要求您登入 Google 帳號進行安全性授權同意。

1. 瀏覽器安全核准

安全機制

Google API 串接極為嚴格，程式採用了一次驗證、本地存儲、且必須經老師點擊核准的安全設計。

避坑指南

雲端自動化開發最常見的兩大巨坑

在研習或實作時，如果遇到 API 阻礙，請參考以下除錯心法，一秒自救：

1. 憑證無效 (invalid_grant)
顯示權限不足。解法：刪除目錄下的 token.json 檔，重啟程式重新勾選所有需要的 API 權限即可。
2. Port 8080 佔用衝突
Windows 系統 8080 常被其他服務佔用。解法：在 Python 驗證處，將 port 改為 0，讓系統自動分配隨機埠口。
