實戰六・萬能插頭

MCP 雲端服務與工具串接

在本單元中，我們將解密 MCP 通用連接埠協定，打破「本地與雲端」的軟體隔閡，讓 AI 特務能操作雲端 Google 試算表與自動傳送完工通知！

1 / 10

行政痛點

AI 特務被關在沙盒中，無法讀取雲端資料？

AI 大腦無法得知網路上公用的 Google 試算表內容，也沒辦法傳送 LINE 訊息通知您的手機，每次都得手動下載上傳？

傳統痛點
AI 只能處理本地檔案。需要操作雲端資料時，老師必須先下載為 Excel，再丟給 AI 處理，完成後手動上傳。
MCP 外掛
為 AI 大腦裝上「USB通用插槽」，它就能直接寫入雲端 Sheets、直接發送 LINE/Google Chat 完工警報。

原理解密

大腦、通用 USB 插槽與外掛手肢

MCP 是由 Anthropic 提出的開源協定，它規範了 AI 大腦如何向外掛伺服器獲取「工具清單」並調用工具的機制。

1. AI 大腦 (Model)
Gemini/Claude 主腦，本身沒有手腳，不知道如何操作雲端 API。
↕ MCP 協定 (萬能插頭)
2. MCP Server (工具外肢)
Sheets/Webhook 連接器，向 AI 宣告它擁有的工具（如：寫入雲端）。

觀念釐清

有了 MCP，為什麼還是需要去 Google Cloud 設定？

MCP 是「讀卡機」，而 Google Cloud 控制台核發的 credentials.json 是「感應門禁卡」，兩者必須搭配才能使用。

🔌 MCP 伺服器 (讀卡機)

幫 AI 裝上萬能轉接頭與工具清單，但本身並沒有登入您個人帳號的權限。

💳 credentials.json (感應門禁卡)

透過 Google Cloud Console 申請的許可證，感應通過後才能允許 MCP 讀寫您的雲端硬碟。

自主決策

AI 是如何決定在何時使用什麼工具？

AI 讀取 MCP 伺服器的說明書後，在對話思考 (Thought) 中，會根據您的指令自主判斷要調用哪個 API 與傳送哪些參數。

1. 讀取說明書：大腦載入 read_spreadsheet、update_spreadsheet 等工具宣告。

2. 意圖對齊：使用者說：「請把新成績寫入雲端...」，AI 判斷這需要調用寫入工具。

3. 自主參數填寫：AI 自主生成 API 請求：update_spreadsheet_values(spreadsheetId='...', range='...', values=[...])。

大對決

MCP 工具連動 vs 傳統寫死 API 程式

傳統寫死程式靈活性極低且開發繁瑣；MCP 的工具是「即插即用」，AI 自主決策，大大釋放了應用邊界。

比較維度 傳統寫死 API (Hard-coded) MCP 服務連動 (Model Context)
決策核心 工程師（用 Code 寫死執行順序） AI 自身（根據 Tool 說明書自主決策）
靈活彈性 極低，API 規格一改程式就必須重寫 極高，伺服器隨插即用，大腦自動適應
安全防禦 背景執行無感，容易有隱蔽漏洞 每次呼叫必跳出 Approve 確認框供人類審核

必裝推薦

適合中小學教師的實用 MCP 工具包

將這三個工具註冊到您的特務大腦中，打通最常碰到的行政與通訊限制：

google-sheets
雲端 Sheets 編輯器。讓 AI 大腦直接取得雲端表格的讀寫權限，即時登錄成績與出勤。
chat-webhook
自動訊息警報。在 AI 跑完耗時的大批次行政任務後，自動發送完工通知到老師的手機上。
filesystem
安全沙盒防護。限制 AI 只能讀寫特定的安全專案資料夾，絕不能亂動 C 槽系統檔案。

實作一

解密 config.json 中的 MCP 註冊配置

老師可以使用 Node.js 執行器直接啟用 Google 官方的試算表 MCP Server，不需安裝繁瑣的資料庫設定。

"google-sheets": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-google-sheets"]
}

說明：npx -y 會自動從雲端套件庫安全下載並連線上網，大腦瞬間學會如何讀寫雲端 Google 試算表！

實作二

完工自動發送 Google Chat 訊息工作流

在 Settings 中註冊完工 Webhook 通訊 MCP 腳本，隨後下達任務指令：

1. 指令與工作
老師命令 AI 分析成績。AI 默默跑 Python 計算分析完成。
↓ AI 自主偵測工具
2. 觸發 Webhook MCP
AI 呼叫 send_chat_notification 工具，手機群組瞬間彈出工作回報！

使用限制

MCP 使用限制與安全把關機制

享受自動化聯動的便利之餘，老師在實務上必須注意以下安全防禦限制：

Approve 安全鎖
AI 每呼叫一次 MCP 工具（如寫入雲端或讀本機），都必須由老師點擊核准，防止惡意刪除。
環境依賴限制
多數 MCP 需要本機 Node.js 或 Python 的後台執行環境。學校學術網路 TANet 防火牆有時會阻擋 npm/npx 聯網安裝。
