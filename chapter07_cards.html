實戰四：課本教材自動生成 Google 表單測驗

扮演數位教學助理，讀取課文文字，自動設計選擇題與正確答案，呼叫 API 建立 Google 表單測驗，自動配分並打亂選項。

輸入：課本教材.txt (段落課文)
大腦命題並寫碼
本機：GCP API 授權 (credentials.json)
一鍵呼叫 API
雲端：Google 測驗表單 (自動配分與隨機排序)

傳統痛點

設計線上測驗的三大繁瑣關卡

傳統上手動建立 Google 表單測驗耗時耗力，容易打錯字且選項配分設定非常繁瑣。

關卡 1：手動剪貼折磨 —— 必須在課文與瀏覽器視窗間來回切換，手動複製題目和選項。

關卡 2：重複設定配分 —— 每一題都要點選「答案金鑰」，輸入分數並指定正確選項，耗時枯燥。

關卡 3：隨機排序繁瑣 —— 必須手動點選每題右下角，勾選「隨機排列選項順序」，操作繁雜。

觀念解密

什麼是 API？認識「金鑰三兄弟」

瞭解 API 的「點餐窗口」白話比喻，以及 credentials.json 與 token.json 的授權分工關係。

🍽️ API 點餐窗口比喻

• 網頁介面 (UI)：餐廳用餐區（人類看畫面）
• Google 伺服器：餐廳廚房（儲存檔案與表單）
• API 窗口：牆上點餐口（傳遞 JSON 資料紙條）

🔑 授權「金鑰三兄弟」

1. credentials.json：軟體身份證（證明程式合規）
2. 瀏覽器授權：主人（您）親自簽名同意
3. token.json：正式感應鑰匙（不需重複登入）

準備工作

教材課文與憑證檔案準備

在 `cards/ch7表單/` 中需要準備好教材檔案，並在此處放入從 Google 下載的憑證金鑰。

課本教材.txt

教材來源範文

包含自然科「水的變化與循環」課文段落，提供 Agent 做為生成 5 題測驗的來源素材。

credentials.json

OAuth 金鑰檔案

從 Google Cloud 平台下載的用戶端金鑰，用於讓本機 Python 取得安全授權。

權限五部曲

GCP 雲端授權與憑證下載

若要在本地端控制 Google 表單，必須先至 Google Cloud Platform 開啟相關 API 與權限。

1. 建立專案
在 GCP 建立 AIAgent-Quiz-Generator 專案。
2. 啟用 API
搜尋並啟用 Google Forms API 和 Google Drive API。
3. 同意畫面
設定 OAuth 同意畫面，並在測試使用者中加入您的 Gmail。
4. 下載憑證
建立桌面版用戶端金鑰，下載並命名為 credentials.json 放入目錄。

核心 Prompt

一鍵發送：啟動表單自動命題

複製首頁一鍵複製工具中的 Prompt 貼給 Antigravity，啟動 AI Agent 本地出題與 API 發送程序。

1. 分析教材：讀取並解析 `課本教材.txt`，設計 5 題配分 20 分的選擇題。

2. 寫 Python 調用 API：在背景寫 Python 調用 Google Forms SDK。

3. 測驗與打亂設定：程式自動開啟 `isQuiz: True`，並對選項隨機排序。

4. 回報編輯與填寫網址：執行完成後，自動輸出老師可編輯的與學生填答的兩個網址。

表單生成與本地授權流程

當您核准代碼後，Agent 核心會遵循 ReAct 循環，在背景自動進行「規劃-行動-觀察」來推動本機 Python 與 Google 雲端通訊。

降維打擊

為什麼網頁版聊天室無法幫您建表單？

網頁版 ChatGPT 或 Gemini 無法存取本地憑證，更無法在您的電腦啟動瀏覽器完成 OAuth 授權。

傳統網頁 Chatbox
只能給您題目文字，您必須手動一題一題複製，並重覆點擊網頁設定答案與分數。無法與您的雲端硬碟聯動。
本地端 Agent
讀取本地 credentials.json，拉起瀏覽器完成授權。直接用 Python 串接 Google 雲端建立測驗，節省 95% 手動時間。

安全機制

安全防線與 token.json 機制

GCP 認證安全非常嚴格，程式採用一次授權、本地儲存的 token 安全方案。

本機安全儲存
第一次在瀏覽器點選同意後，產生的 token.json 留在您的本機上。不需要把密碼或 API Key 寫死在程式碼中。
人機審查
Agent 在背景所編寫的 Forms API Python 代碼，在執行前都必須經過老師的「核准」點擊，完全符合安全防禦。
