為什麼先講 GitHub

老師要先知道自己在發布什麼

GitHub 不僅是「網路上的作品資料夾」，更像是一個線上的教材備份庫。它能妥善儲存每次修改的歷史版本，還能一鍵將 HTML 檔案發布成網址，讓學生不需下載就能在任何裝置上直接開啟作答。

GitHub
線上的教材備份庫與作品資料夾，可以放 HTML、小測驗與說明文件，並留存歷史版本。
Repository
一個教材專案資料夾，例如 grade3-chinese-quiz。
GitHub Pages
把 HTML 發布成學生可以點開的網址。

認識版本控制

Git 到底是什麼？檔案的時光機

Git 是一個安裝在您電腦上的版本控制軟體。它就像是一台「版本照相機」，當您修改教材時，隨時可以拍照存檔，並能隨時搭乘時光機還原到之前的任何狀態。

📸 版本相機
幫每次修改的教材記錄變更歷程（Commit），留下當時的「檔案快照」。
⏳ 安全時光機
當檔案不小心改壞、公式寫錯或想反悔時，隨時能一鍵還原到過去的正常版本。
📁 乾淨資料夾
不需要手動備份 教案_v1、教案_v2_最終版，資料夾內永遠只有一份最乾淨的檔案。
🧠 本地隱藏腦
完全離線可用，所有歷史版本紀錄全部安全地儲存在專案內隱藏的 .git 資料夾中。

Git 與 GitHub 關聯

Git 是紀錄工具，GitHub 是發布平台

Git 負責在「本機」記錄每次教材的修改，就像拍照存檔；GitHub 則是「雲端」作品庫，負責將教材發布成網址供學生存取。
💡 貼心提醒：這些 Git 指令現在不用記，AI Agent 都可以幫您執行！

本機 Git
記錄工具。利用 git add 與 git commit 在電腦中留下版本。
雲端 GitHub
存放空間。接收本機推送過去的檔案，作為發布的起點。
如何傳送？
關聯橋樑。使用 git push 指令將本機的版本紀錄送到 GitHub。
發布成果
開啟 Pages。在 GitHub 設定中開啟 Pages，即可產生學生作答的網頁網址。

GitHub 核心定位

為什麼免費？它還能做什麼？

GitHub 採用「個人公開免費，企業隱密收費」模式，為教師提供穩定的免費發布平台，更擁有豐富的 AI 與自動化擴充服務。

個人免費
公開的教材專案 100% 免費，背後由微軟支援，穩定且安全。
AI Copilot
全球最強 AI 程式助理，通過審核的教師與學生可「免費」使用。
Pages 網頁
把 HTML 程式碼一鍵變成公開網址，直接提供給學生作答。
自動化 Actions
自動化流水線，幫您在背景自動檢查、轉換格式與更新網站。

驅動指令

從零發布的 Agent Prompt

老師只要把自己的 GitHub repo 網址換進去，剩下交給 Agent 整理檔案與執行 Git。

請協助我從零開始完成 GitHub 發布。 1. 檢查三個實戰二產物是否存在。 2. 建立 publish/grade3-chinese-quiz/。 3. 將互動小測驗複製為 index.html。 4. 撰寫 README.md。 5. 執行 git init / add / commit / branch。 6. 連接 GitHub repository。 7. push 到 main。 8. 若需要登入或 Token，請暫停並用白話告訴我。

安全提醒

Token 和登入權限要由老師把關

GitHub 發布會碰到帳號權限。Agent 可以協助操作，但不能替老師保管密碼或 Token。

可以
讓 Agent 提醒現在要登入、要授權、要開啟 Pages。
不可以
把 Token 寫進 Markdown、HTML、README 或聊天紀錄中保存。
要確認
發布內容不能含學生個資或不該公開的答案資料。
